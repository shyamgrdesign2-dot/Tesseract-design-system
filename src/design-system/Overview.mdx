# Overview

**Tesseract** is the component library for the **TatvaPractice ecosystem**, one
coherent set of building blocks shared across every product. Each component is
**first-party, configuration-driven, and reusable anywhere**, so it drops into any
screen and reads as the same surface.

This page is the entry point for the team and the community: it states the
principles, and **[Component Standards](?path=/docs/design-system-component-standards--docs)**
is the contract every component must satisfy.

---

## Principles

1. **Reusable components.** Every component is first-party and built to drop into
 any product or screen without rework, so the whole ecosystem stays consistent by
 default.

2. **Tesseract tokens only.** Color, radius, spacing come from `var(--tesseract-*)` design
 tokens, never raw hex. Swap the token layer, retheme the whole system.

3. **Tesseract Icons only, no inline SVGs.** Every glyph comes from the **Tesseract Icons** CDN,
 the single source of truth for icons. Components never hand-write an `<svg>` (the
 only exceptions are stateful visuals like the sort indicator and `AnimatedGrid`).

4. **Configuration-driven.** Components are described by data, a column config,
 a tabs array, a filter-groups config, not by bespoke markup per screen. Add
 a column, add a tab, change a filter: edit config, not code.

5. **Everything in Controls.** Every variant, state, and per-part option a
 component supports is a Storybook control. If you can't drive it from the
 Controls panel, it isn't done.

6. **Responsive & accessible by default.** Horizontal scroll over squish, sticky
 affordances with shadows, truncation + tooltip for overflow, keyboard nav,
 ARIA roles, focus-visible rings.

7. **Theme from a seed.** `createTheme({ brand })` derives the full token set
 (50→900 ramps, component radii) from one colour; `TPThemeProvider` applies it
 with no component changes. See **Foundations → Theme Provider**.

8. **Tracking is opt-in & free.** Interactive components emit semantic events to
 the nearest `TPAnalyticsProvider` (`Button.track`, `DataTable.analyticsId`, …).
 No provider → a no-op, zero overhead.

---

## How it's organized

- **Atoms**, primitives: `Button`, `Badge`/`Chip`, `Checkbox`, `Radio`,
 `Toggle`, `Slider`, `Input`, `Tooltip`, `Divider`, `Skeleton`,
 `LoadingIndicator`, `TPIcon` + `IconPicker`.
- **Molecules**, composed surfaces: `DataTable` (+ `DataCell`/`TableActions`),
 `Tabs`, `Dropdown`, `Toast`, `Filter`, `ConfirmDialog`, `Dialog`, `DatePicker`
 (single · range · month · year · time · datetime), `Accordion`, `HeroBanner`.
- **Foundations**, the design tokens in `src/tesseract-tokens.css` (colors, typography,
 radius, shadow, spacing, gradients), shown under **Design System → Foundations**,
 the **Theme Provider** (`createTheme`), and the **CDN icon library**.

---

## The icon system

Every icon comes from one place, the **Tesseract Icons** CDN. It holds the full library,
and each icon is rendered mask-themed, so it tints to `currentColor` and inherits
size from its context. Components never hand-write an `<svg>`.

- **In components**, icons are wired in directly and can swap style for state
 (for example `linear → bulk` when active).
- **In your own screens**, any icon slot opens the searchable **Tesseract Icons** picker,
 flag an argType `tpIcon: true` and it becomes pickable.
